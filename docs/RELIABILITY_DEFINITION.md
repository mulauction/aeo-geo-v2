# Reliability Definition & Copy Standardization

## Reliability 레벨 정의

Reliability는 리포트의 신뢰도를 나타내는 지표로, 다음 3가지 레벨로 구분됩니다:

### 1. 높음 (High)
- **판정 규칙**: Brand, Content, URL 3개 항목 모두 측정 완료 + 최소 기준 충족
- **조건**:
  - `branding` 점수가 실측 상태 (null/undefined/NaN 아님)
  - `contentStructureV2` 점수가 실측 상태 + evidence 배열 존재 (length > 0)
  - `urlStructureV1` 점수가 실측 상태
  - 각 점수가 최소 기준(threshold)을 충족 (score >= 0)
- **사용자 문구**: "높음"

### 2. 보통 (Mid)
- **판정 규칙**: 2개 항목 측정 완료
- **조건**: Brand, Content, URL 중 2개 항목이 측정 완료된 상태
- **사용자 문구**: "보통" 또는 "중간"

### 3. 낮음 (Low)
- **판정 규칙**: 1개 이하 항목만 측정 완료 또는 리포트 없음
- **조건**:
  - Brand, Content, URL 중 1개 이하만 측정 완료
  - 또는 리포트 로드 실패/근거 접근 불가
- **사용자 문구**: "낮음"

### 4. 측정 필요 (Unknown)
- **판정 규칙**: 입력 데이터가 거의 없거나 리포트가 없는 경우
- **조건**:
  - 모든 항목 미측정 (n === 0) + 입력 데이터 없음
  - 또는 리포트 로드 실패
- **사용자 문구**: "측정 필요"

## 레벨 판정 규칙 (측정 여부 카운트 기반)

Reliability 레벨은 다음 3개 항목의 측정 여부를 카운트하여 결정됩니다:

1. **BRAND**: `branding` 점수가 실측 상태인지 여부
2. **CONTENT**: `contentStructureV2` 점수가 실측 상태이고 evidence 배열이 존재하는지 여부
3. **URL**: `urlStructureV1` 점수가 실측 상태인지 여부

측정 완료된 항목 개수(n)에 따라:
- n === 3 → 높음
- n === 2 → 보통
- n <= 1 → 낮음 (단, n === 0이고 입력 데이터도 없으면 "측정 필요")

## 사용자에게 보여줄 문구 원칙

### 기본 원칙: (라벨 + 짧은 이유 1개 + 행동 1개) 한 줄

Reliability 배지 옆에 표시되는 한 줄 안내 문구는 다음 형식을 따릅니다:

```
[라벨] [짧은 이유 1개] · [행동 1개]
```

### 라벨
- 높음: "높음"
- 보통: "보통" 또는 "중간"
- 낮음: "낮음"
- 측정 필요: "측정 필요"

### 이유 문구 표준화

이유(reasons)는 다음 순서로 고정하여 표시합니다:
1. **BRAND** 측정 상태
2. **CONTENT** 측정 상태
3. **URL** 측정 상태
4. **리포트 없음** (해당 시)

각 이유는 짧고 일관된 톤으로 통일합니다:
- 동일 조사: "측정됨" / "미측정"
- 동일 종결: "·" (중점)으로 구분

### 예시

**높음 케이스:**
```
높음 · BRAND 측정됨 · CONTENT 측정됨 · URL 측정됨 · 3개 항목 모두 측정 + 최소 기준 충족
```

**보통 케이스:**
```
보통 · BRAND 측정됨 · CONTENT 측정됨 · URL 미측정
```

**낮음 케이스:**
```
낮음 · BRAND 미측정 · CONTENT 측정됨 · URL 미측정
```

**측정 필요 케이스:**
```
측정 필요 · BRAND 미측정 · CONTENT 미측정 · URL 미측정
```

**리포트 없음 케이스:**
```
리포트 없음 · 리포트를 불러올 수 없습니다
```

## 리포트 없음(no-report) 케이스 처리

리포트 없음 케이스는 "측정 필요"와 혼동되지 않도록 명확히 구분합니다:

- **리포트 없음**: 리포트 자체를 불러올 수 없는 경우
  - 문구: "리포트 없음 · 리포트를 불러올 수 없습니다"
  - 또는: "리포트 로드 실패 · 리포트를 불러오지 못했어요"

- **측정 필요**: 리포트는 있지만 모든 항목이 미측정인 경우
  - 문구: "측정 필요 · BRAND 미측정 · CONTENT 미측정 · URL 미측정"

## WHY 패널/Action line과의 정합성

Reliability 배지의 한 줄 안내 문구는 WHY 패널의 reasons 및 Action line과 충돌하지 않아야 합니다:

- Reliability 배지: 측정 상태 요약 (BRAND/CONTENT/URL 측정 여부)
- WHY 패널: 구체적인 부족한 근거나 개선점
- Action line: 실행 가능한 행동 제안

세 가지가 서로 일관되게 표시되어야 하며, 같은 정보를 다른 방식으로 표현하지 않아야 합니다.
